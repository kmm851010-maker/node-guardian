------------------------------
제작자 배경 (대본에 반영)
------------------------------

- 4년째 파이 노드 2대 운영 중
- 주중에는 타지 생활을 해서 집 PC를 자주 못 봄
- 바쁜 사이에 PC가 재부팅되어 노드가 꺼져 있는 걸 뒤늦게 발견하는 일 반복
- 이 불편함을 해결하려고 직접 만든 툴
- 광고나 수익 목적이 아닌, 같은 고민을 가진 파이오니어를 돕고 싶어서 제작


------------------------------
앱 구성
------------------------------

NodeGuardian (PC 프로그램):
  - PC에 설치하는 프로그램
  - 설치 후 백그라운드에서 자동 실행 (트레이 아이콘)
  - PC 켜져 있으면 알아서 노드 상태를 서버에 전송
  - 별도로 켜거나 신경 쓸 필요 없음
  - 다운로드: GitHub releases 또는 pilink.vercel.app (PC 브라우저에서만 다운로드 가능)

LinkPi 웹앱:
  - 파이브라우저에서 접속하는 웹앱
  - 별도 설치 없음
  - Pi 계정으로 바로 로그인 (별도 회원가입 없음)
  - NodeGuardian이 보낸 데이터를 시각화해서 보여줌
  - 접속 주소: pilink.vercel.app (파이브라우저에서만 로그인 및 주요 기능 이용 가능)


------------------------------
핵심 강조 메시지 (영상 전체에 반복)
------------------------------

1. 한 번 설치/설정하면 끝 — 이후 방치해도 됨
2. 알림(텔레그램)이 왔을 때만 파이브라우저로 접속해서 확인
3. 확인 후 조치가 필요하면 PC 원격으로 바로 처리
4. 완전 무료 (핵심 기능 전부)


------------------------------
주요 기능 설명
------------------------------

1. 노드 상태 모니터링
   - 정상 / 경고 / 중단 / 오프라인 4단계 표시
   - 마지막 신호 수신 시간 표시
   - 7일 / 30일 가동률 퍼센트 (색상 코딩: 99%↑ 초록, 95%↑ 노랑, 이하 빨강)
   - 요일별 상태 타임라인 시각화

2. 이벤트 기록
   - 노드가 언제 꺼졌는지, 언제 복구됐는지 기록
   - 정보 / 경고 / 위험 / 복구 구분
   - 탭하면 상세 내용 확인 가능

3. 텔레그램 알림 (유일한 알림 수단)
   - Pi Browser는 웹 푸시 알림 미지원 → 텔레그램으로 대체
   - 연동 방법: 텔레그램에서 @serge_node_guardian_bot 검색 → /start 입력 → Chat ID 확인 → 앱에 입력
   - 노드 이상 발생 시 즉시 텔레그램 알림 수신

4. 커뮤니티 / Q&A
   - 노드 운영자끼리 노하우 공유 게시판
   - Q&A에서 답변 채택 시 랭킹 점수 획득
   - 현재 초기 단계, 사용자 늘어나면 정보 교류 활성화 기대
   - 이용하려면 프리미엄 구독 필요 (1 Pi/월)

5. 주간 랭킹
   - 커뮤니티/Q&A 활동 + 출석 XP 합산 점수로 순위 결정
   - 매주 1~10위에게 Pi 상금 지급
   - 랭킹 탭에서 확인 가능

6. 출석 체크 & 레벨 시스템
   - 매일 출석 체크 시 +10 XP
   - 레벨: 씨앗(0) → 새싹(100) → 나무(300) → 노드(700) → 마스터(1500)
   - 주간 XP는 주간 랭킹 활동점수에 반영

7. 노드 고유번호 등록
   - Pi 앱에서 노드 고유번호 확인 후 입력
   - 파이 블록체인 익스플로러에서 내 노드 랭킹 바로 확인 가능

8. 프리미엄 구독 (선택사항)
   - 1 Pi / 월
   - 커뮤니티, Q&A 참여 등 추가 기능
   - 핵심 기능(모니터링, 알림, 이벤트 기록)은 무료


------------------------------
영상 구조 (10분 내외)
------------------------------

① 훅 (0~30초)
목적: 시청자가 끝까지 볼 이유를 만들기
내용: 노드 꺼진 거 모르고 지나친 경험 공감 → "이제 안 확인하셔도 됩니다" 유도

② 제작자 소개 + 문제 공감 (30초~1분30초)
목적: 신뢰감 형성, 공감대 구축
내용: 4년 노드 운영자, 타지 생활, 뒤늦게 꺼진 것 발견, 직접 만들게 된 이유

③ 앱 구조 설명 (1분30초~2분30초)
목적: NodeGuardian + LinkPi 두 가지 구성 이해시키기
내용: PC 프로그램 + 웹앱 구조, 어떻게 연동되는지 간단히

④ NodeGuardian 설치 시연 (2분30초~4분)
목적: 설치가 쉽고 방치해도 된다는 것 보여주기
내용: 다운로드 → 설치 → 트레이 아이콘 → "이제 신경 안 써도 됩니다"

⑤ LinkPi 접속 + 주요 기능 시연 (4분~7분)
목적: 핵심 기능 직접 보여주기
내용: 파이브라우저 접속 → 로그인 → 대시보드(노드 상태/가동률/이벤트) → 텔레그램 알림 설정 순서로 시연

⑥ 실제 사용 시나리오 (7분~8분30초)
목적: "이렇게 쓰면 된다" 구체적 그림 그려주기
내용: 텔레그램 알림 수신 → 파이브라우저로 원인 확인 → 별거 아니면 닫기 / 조치 필요하면 PC 원격으로 처리

⑦ 추가 기능 소개 (8분30초~9분30초)
목적: 모니터링 외에도 가치 있다는 것 보여주기
내용: 커뮤니티/Q&A, 주간 랭킹 상금, 출석 XP/레벨, 노드 고유번호 등록

⑧ 마무리 CTA (9분30초~10분)
목적: 시청자가 바로 써보도록 유도
내용: "무료입니다, 지금 파이브라우저에서 접속해보세요" → 앱 주소 자막 → 공유 요청


------------------------------
톤 & 스타일
------------------------------

- 친근하고 편안한 말투, 전문적이지만 어렵지 않게
- 같은 파이오니어로서 경험을 공유하는 느낌 (홍보 느낌보다는 추천하는 느낌)
- 과장 없이 담백하고 솔직하게
- 시청자 호칭: "파이오니어 여러분" 또는 "노드 운영하시는 분들"
- 기술 용어는 쉽게 풀어서 설명
- 핵심 메시지 중간중간 자연스럽게 반복:
    "한 번 설정하면 끝이에요"
    "알림 올 때만 확인하시면 됩니다"
    "완전 무료입니다"
- 시연 장면은 화면을 보며 설명하듯 자연스럽게
- 마무리는 강요 없이 편하게 권유


------------------------------
주의사항
------------------------------

- 알림은 텔레그램만 지원 (푸시 알림 언급 금지)
- 파이브라우저에서만 로그인 및 주요 기능 이용 가능 (일반 브라우저에서는 모니터링 화면만 조회 가능)
- NodeGuardian 다운로드는 PC 브라우저에서만 가능
- 파이 네트워크 기초 설명 불필요 (시청자는 이미 파이오니어)
- 타겟: 노드를 운영 중인 파이오니어 (노드 없는 파이오니어 대상 아님)
- 프리미엄은 선택사항임을 명확히 (무료로도 핵심 기능 모두 사용 가능)
- 커뮤니티는 현재 초기 단계임을 솔직하게 언급
